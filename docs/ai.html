Use Abrantes with AI

How to generate Abrantes experiments with AI tools and models, based on practical tests.

Experiment guide Open SKILL.md

Important disclaimer

This workflow is experimental. Do not rely only on AI to create experiments.

  • Always review generated code before using it.
  • Always test variants and tracking before launch.
  • Treat model output as draft code, not as production-ready code.

Recommended workflow

  1. Use a coding-focused AI tool that can read files in a local Abrantes repo clone.
  2. Share your target URL and experiment hypothesis.
  3. Provide SKILL.md (or reference the Abrantes docs URL if upload is not supported).
  4. Ask for a complete experiment with variant logic, render, persist, and tracking calls.
  5. Run and validate the output manually before launch.

Tools that failed in tests

These results were noted on 2026-02-28 and may change as tools improve.

  • GitHub + Claude Haiku 4.5 (online): could not access the tested page reliably.
  • Gemini online: hallucinated the implementation.
  • Mistral Le Chat: hallucinated the implementation.
  • Grok 4.20 (beta): many selector and implementation mistakes.
  • Qwen 3.5 plus reasoning: made up behavior and struggled with add2dataLayer.
  • DeepSeek: generated invented behavior and ignored the real page context.
  • Google NotebookLM: ignored uploaded SKILL.md guidance.
  • ChatGPT Mac app (free plan): could not access tested URLs directly in this workflow.

Final recommendations

  • Evaluate a tool + model pair, not only the model alone.
  • Desktop coding tools usually perform better because they can inspect local repo files.
  • If your tool cannot access the tested page, experiment quality drops significantly.
  • Before launch, validate selectors, assignments, rendering, persistence, and tracking events.
